# AI女友 GDD

## 定位

`AI女友` 是一款单机优先的剧情陪伴游戏。玩家每天短时间进入共同房间，和角色林栖聊天、推进章节剧情、完成小约定、解锁共同回忆，让角色的语气、情绪和房间状态随关系变化。

## 目标平台

- 当前原型：PWA，Android 浏览器和 macOS 浏览器可运行。
- Android 正式版：Capacitor 打包 WebView APK。
- macOS 正式版：Tauri 或 Electron 打包桌面 App。

## 核心体验

- 角色像住在游戏中的人，而不是工具型聊天框。
- 玩家和角色之间有长期痕迹：记忆、日记、约定、房间物品。
- 单机可用，联网 AI 只是增强项。
- 每天 5-15 分钟即可获得反馈。

## 角色

名字：林栖

性格：温柔、敏感、略认真，不完全顺从。她会关心玩家，也会保留自己的喜好和小情绪。

偏好：夜晚、旧唱片、热牛奶、手写日记、安静的房间。

边界：不鼓励玩家逃避现实，不制造强依赖，不替代医疗或心理建议。

## 当前系统

### 陪伴聊天

玩家可以自由输入一句话。原型用关键词规则匹配回复，并同步改变角色情绪、关系数值和记忆。

后续接入大模型时，游戏仍保留本地记忆、角色设定和安全边界，避免完全交给模型自由发挥。

### 关系阶段

内部使用亲密度驱动 5 个阶段：

- 相遇
- 熟悉
- 靠近
- 依赖
- 共生

阶段不作为硬数值展示，而是通过顶部状态、回复文本和日记体现。

### 情绪与状态

当前有亲密、信任、压力三个可见状态。正式版建议将部分数值隐藏，只保留角色行为表现。

### 共同房间

房间是主界面。玩家通过互动解锁唱片、热牛奶、合照等物品。每个物品会触发独立对话，形成共同生活感。

### 轻任务和约定

当前包含一个睡前分享约定。正式版可以扩展为：

- 明天提醒喝水
- 周五一起听歌
- 每晚说一句晚安
- 周末整理房间
- 一起记录开心小事

## MVP 验收标准

- 可在无构建工具情况下打开运行。
- 能在移动端竖屏正常操作。
- 能保存关系状态、记忆、日记和约定。
- 至少 6 个互动动作。
- 至少 5 类聊天关键词反馈。
- 具备离线缓存能力。

## 下一阶段

1. 增加 30 个日常事件和 10 个剧情节点。
2. 增加本地角色记忆编辑器。
3. 加入 Live2D 或序列帧表情动画。
4. 接入可选大模型服务。
5. 打包 Android/macOS 安装版。
### 剧情章节

原型加入章节剧情卡片，每个章节包含一段当前情境和两个选择。玩家选择后会改变亲密、信任、压力、记忆和日记，并推进下一段剧情。

当前章节：

- 雨夜来信
- 桌边的灯
- 靠近玩耍
- 短暂僵住
- 周五的约定

正式版建议每个关系阶段至少配置 8-12 个剧情节点，并让节点受玩家历史记忆影响。

### 2D 角色表现

当前版本使用 2D 视觉小说角色主视觉，并加入男主和林栖同屏的双人关系 CG。前端实现轻量动态表现：

- 呼吸浮动
- 轻微身体摆动
- 自动眨眼
- 情绪光效变化
- 情绪文字状态同步
- 双人互动状态标签
- 拥抱、亲吻、闹脾气、玩游戏等关系事件

正式版建议升级为 Live2D 或 Spine：拆分头发、脸、眼睛、嘴、身体、手臂等图层，加入表情切换、口型同步和剧情动作。

### 双人互动

男主不再只作为输入框背后的玩家存在，而是作为游戏关系的一部分出现。当前可触发：

- 拥抱：降低压力，提高亲密。
- 亲吻：增加亲密和信任，触发害羞状态。
- 闹脾气：短期增加压力，但推动关系修复剧情。
- 玩游戏：降低压力，增加轻松感，并生成下一次小游戏约定。

## Skill Review Notes

按照 `2d-game-designer`、`2d-game-prototyper` 和 `playtest-reviewer` 的检查项，当前版本需要优先保证：

- 一屏内能看懂目标：陪伴林栖并推动关系。
- 第一交互明显：剧情选择和双人互动直接浮在画面上。
- 反馈即时：对白、情绪、记忆、约定、状态同步变化。
- 原型测试可重复：静态检查和运行时烟测必须覆盖核心按钮。
- 真机试玩重点：移动端遮挡、触摸区域、缓存更新和沉浸感。
