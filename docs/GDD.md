# AI女友 GDD

## 定位

`AI女友` 是一款单机优先的 2D 探索陪伴游戏。整体结构参考 `原神` 这类沉浸式二次元游戏的“走路、探索、触发事件、推进任务”节奏，但以轻量 2D 视角实现，适合 Android 手机和 macOS 浏览器。

## 目标平台

- 当前原型：PWA，可在 Android 浏览器和 macOS 浏览器运行。
- Android 正式版：WebView APK。
- macOS 正式版：Tauri 或 Electron。

## 核心体验

- 玩家先操作角色移动，再通过靠近和互动推进关系。
- 世界要像一个能逛的空间，不是单独的对话框。
- 角色关系通过走位、停留、共同拍照、完成约会任务留下痕迹。
- 单机可用，联网 AI 只是增强项。
- 每次进入游戏都能在上次进度上继续走一段。

## 角色

名字：林栖

性格：温柔、敏感、略认真，不完全顺从。她会带路，也会等玩家走到她身边。

偏好：夜晚、旧唱片、热牛奶、拍照、安静的街区、公园长椅。

边界：不鼓励玩家逃避现实，不制造强依赖，不替代医疗或心理建议。

## 当前系统

### 探索移动

玩家可以用键盘、摇杆或点击地面移动。镜头会跟随角色，形成连续的 2D 场景体验。

### 任务推进

当前原型采用顺序任务链：

1. 走到林栖身边
2. 穿过门廊去街区
3. 捡起纸飞机
4. 和她坐到长椅上
5. 在喷泉前拍照
6. 把照片挂回房间

### 关系成长

关系主要通过羁绊值、记忆、照片、动作解锁推进。当前不依赖复杂数值，而是让玩家通过完成任务看到角色行为变化。

### 技能解锁

- 冲刺：在街区找到纸飞机后解锁
- 拍照：在公园完成照片任务后解锁

### 记忆系统

玩家每完成一个任务，都会留下新的记忆文本。后续可扩展为照片墙、路线记录和可回访地点。

## MVP 验收标准

- 一打开就是可移动的 2D 场景，而不是聊天窗口。
- 能在移动端竖屏正常操作。
- 能保存任务进度、羁绊和记忆。
- 至少 6 个探索/互动节点。
- 至少 2 个能力解锁点。
- 具备离线缓存能力。

## 下一阶段

1. 增加更大的连续地图和更多可进入区域。
2. 加入更明显的 NPC 路径引导和小地图。
3. 增加更多约会事件和探索收集。
4. 加入更细的表情、动作和摄影反馈。
5. 打包 Android/macOS 安装版。

## Skill Review Notes

按照 `2d-game-designer`、`ux-game-designer`、`level-designer` 和 `playtest-reviewer` 的检查项，当前版本需要优先保证：

- 一屏内能看懂目标：先走到林栖身边，再一路推进任务。
- 第一交互明显：移动后就能看到目标提示和互动点。
- 反馈即时：羁绊、任务、记忆、能力解锁同步变化。
- 原型测试可重复：静态检查和运行时烟测必须覆盖核心操作。
- 真机试玩重点：移动端遮挡、摇杆手感、镜头跟随和任务提示。
