# AI女友 2D 游戏规格

## Premise

玩家扮演男主，与林栖在同一个 2D 视觉小说场景里相处。游戏重点不是管理数值，而是通过剧情选择、双人互动和短句聊天，让关系在每次进入游戏时留下可见痕迹。

## Target Session

- 平台：Android 手机浏览器、macOS 浏览器
- 输入：点击、触摸、短文本输入
- 单次时长：5-12 分钟
- 核心玩家能力：观察林栖的情绪和上下文，选择合适的互动方式

## Core Loop

1. 观察当前场景、章节和林栖情绪。
2. 在剧情选择、双人互动或自由聊天中选择一个行动。
3. 系统即时反馈：对白、情绪、关系状态、记忆、日记、约定更新。
4. 关系推进到下一段剧情或解锁新的共同痕迹。
5. 玩家决定继续陪伴、修复情绪、制造亲密时刻或结束当天会话。

## Player Verbs

- 选择剧情回应
- 拥抱
- 亲吻
- 闹脾气后修复
- 一起玩游戏
- 听歌
- 做约定
- 送礼物
- 输入一句自由聊天

## First 5 Minutes

1. 玩家打开游戏，看到全屏 2D 双人场景，而不是后台式面板。
2. 顶部 HUD 显示天数、阶段、情绪和当前互动状态。
3. 右侧剧情卡出现第一段章节，玩家做出第一个选择。
4. 底部对白框反馈林栖的回应。
5. 玩家尝试一次双人互动，例如拥抱或亲吻。
6. 记忆、日记或约定被写入，关系状态变化。

## Progression

- 玩家掌握：从随便点按钮，变成根据情绪选择互动。
- 内容解锁：剧情章节、记忆、约定、房间物件。
- 关系成长：相遇、熟悉、靠近、依赖、共生。
- 表达空间：玩家可以通过自由输入塑造关系语气。

## Failure And Recovery

本游戏不设计传统 Game Over。失败表现为压力上升、林栖闹脾气、信任增长变慢或剧情进入修复节点。

恢复方式：

- 解释原因
- 多陪她一会儿
- 拥抱或一起玩游戏降低压力
- 做一个新的约定

## Reward Rules

- 亲密增加：更多靠近、亲吻、共同回忆。
- 信任增加：承认疲惫、解释冲突、认真回应。
- 压力降低：陪伴、玩游戏、听歌、晚安。
- 记忆/日记：让关系留下文本痕迹。
- 约定：形成回访理由。

## UI Flow

启动即进入沉浸式场景：

- 全屏 2D CG 是主体。
- 顶部只保留轻 HUD。
- 右侧浮层承载剧情选择。
- 底部对白框承载当前对话。
- 记忆/日记/约定不作为主界面核心，只作为辅助记录。

## Tuning Risks

- 双人互动如果只换文字，不换 CG，仍会显得静态。
- 剧情选择数量不足会让玩家很快感到重复。
- 自由聊天规则太少时，玩家会感到“假 AI”。
- 移动端可视区域小，剧情卡和对白框容易遮挡角色。

## Prototype Scope

当前原型应优先完成：

- 沉浸式视觉小说布局
- 双人 CG 主画面
- 4 个核心双人互动
- 5 个剧情章节
- 本地存档
- 运行时烟测

下一版再做：

- 拥抱、亲吻、吵架、玩游戏专用 CG
- 分情绪表情图
- 移动端真机试玩调整
- Live2D 或 Spine 拆图动画
