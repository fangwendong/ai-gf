# AI女友 2D 游戏规格

## Premise

玩家扮演男主，在同一张 2D 地图里和林栖探索、约会、拍照、推进任务。目标不是在对话框里“聊完”，而是像 `原神` 那样先走、先看、先靠近，再在场景里触发关系推进。

## Target Session

- 平台：Android 手机浏览器、macOS 浏览器
- 输入：点击、触摸、键盘、触摸摇杆
- 单次时长：5-12 分钟
- 核心玩家能力：观察场景目标、规划路线、靠近交互点、完成约会任务

## Core Loop

1. 观察当前场景、任务目标和最近的互动点。
2. 用移动、摇杆或点击去接近目标。
3. 在目标范围内触发互动，推进任务或解锁能力。
4. 系统即时反馈：羁绊、记忆、能力、提示文本更新。
5. 玩家继续探索下一段路线，或者回到房间整理照片。

## Player Verbs

- 移动
- 靠近
- 互动
- 冲刺
- 拍照
- 记忆回看

## First 5 Minutes

1. 玩家打开游戏，先看到全屏探索地图和当前目标。
2. 第一件事是走到林栖身边，学会移动和靠近。
3. 任务把玩家带出房间，进入街区和公园。
4. 玩家捡起纸飞机，解锁冲刺。
5. 在公园坐下、拍照、回家挂照片，形成第一段完整约会。

## Progression

- 玩家掌握：从乱走，变成会看目标和走位。
- 内容解锁：新区域、任务、记忆、拍照。
- 能力成长：冲刺、拍照、更多互动点。
- 关系成长：羁绊值提升，林栖会更主动带路和靠近。

## Failure And Recovery

本游戏不设计传统 Game Over。失败表现为走错方向、离目标太远、暂时没法互动。

恢复方式：

- 给出明确提示
- 重新靠近目标
- 用冲刺缩短距离
- 继续当前任务，不打断节奏

## Reward Rules

- 完成任务：推进地图和关系。
- 解锁冲刺：扩大可探索节奏。
- 解锁拍照：增加表达和留存。
- 记忆：让关系留下文本痕迹。

## UI Flow

启动即进入沉浸式探索：

- 全屏 2D 地图是主体。
- 顶部只保留轻 HUD。
- 底部保留目标提示和互动按钮。
- 任务面板和对话面板都是辅助层。

## Tuning Risks

- 地图如果过空，会像背景图而不是场景。
- 目标提示如果不够明确，玩家会迷路。
- 手机摇杆如果太小，会影响移动手感。
- 互动点如果过少，探索会很快变成直线任务。

## Prototype Scope

当前原型优先完成：

- 2D 探索地图
- 角色移动与镜头跟随
- 6 个任务节点
- 2 个能力解锁
- 本地存档
- 运行时烟测

下一版再做：

- 更大的连续地图
- 小地图与任务引导
- 更多 NPC 和可交互场景
- 更细的摄影和表情反馈
